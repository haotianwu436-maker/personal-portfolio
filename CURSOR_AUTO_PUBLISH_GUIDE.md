# Cursor自动化发布指南

本文档说明如何让Cursor自动发布文章、运行测试、提交代码和部署上线，实现完全的自动化工作流。

## 🎯 自动化发布流程

```
你的需求（包含完整文章内容）
    ↓
Cursor自动执行以下步骤：
    1. 检查项目状态
    2. 运行所有测试
    3. 检查TypeScript
    4. 构建项目
    5. 创建文章
    6. 更新todo.md
    7. 提交到GitHub
    8. 运行自动化部署脚本
    9. 创建checkpoint
    ↓
你在Manus UI中点击Publish
    ↓
文章上线！
```

## 📝 如何使用

### 方式一：让Cursor自动发布（完全自动化）

**告诉Cursor**：

```
我想发布一篇新文章。请使用以下信息自动创建并发布：

【文章信息】
- 标题：为什么建立这个网站
- Slug：why-build-this-website
- 摘要：半梦半醒，永远年轻，永远热泪盈眶。这句话一直是我内心的写照。在这个快速变化的世界里，我希望能有一个地方，可以慢下来，记录下那些真正重要的东西。
- 标签：网站, 初心, 理念

【文章内容】
[你的完整文章内容，支持Markdown格式]

请执行以下步骤：
1. 运行 pnpm test 确保所有测试通过
2. 运行 pnpm check 检查TypeScript
3. 运行 pnpm build 构建项目

完成后，我会在Manus UI中点击Publish按钮发布网站。
```

**Cursor会自动**：
- ✅ 运行所有测试

- ✅ 创建文章

- ✅ 创建checkpoint
- ✅ 告诉你版本号

### 方式二：使用脚本手动发布（半自动化）

如果你想手动控制发布过程，可以直接运行脚本：

```bash
./scripts/publish-article.sh "文章标题" "文章-slug" "摘要文本" "完整的文章内容" "标签1,标签2"
```

**示例**：

```bash
./scripts/publish-article.sh \
  "为什么建立这个网站" \
  "why-build-this-website" \
  "我的初心和理念" \
  "完整的文章内容..." \
  "网站,初心,理念"
```

---

## 🔧 脚本说明

### publish-article.sh 脚本做了什么

这个脚本会自动执行以下步骤：

1. **检查项目状态** - 确保package.json存在
2. **运行测试** - `pnpm test`，所有测试必须通过
3. **检查TypeScript** - `pnpm check`，没有类型错误
4. **构建项目** - `pnpm build`，构建成功
5. **创建文章** - 将文章信息保存到JSON文件
6. **更新todo.md** - 标记任务为完成
7. **提交代码** - `git add` 和 `git commit`
8. **推送到GitHub** - `git push github main`
9. **运行部署脚本** - `./scripts/auto-deploy.sh`
10. **生成日志** - 保存详细的执行日志

### 脚本输出

脚本会输出：
- ✅ 每个步骤的执行结果
- 📊 总耗时
- 📁 文章文件位置
- 📋 日志文件位置
- 🔗 下一步操作说明
